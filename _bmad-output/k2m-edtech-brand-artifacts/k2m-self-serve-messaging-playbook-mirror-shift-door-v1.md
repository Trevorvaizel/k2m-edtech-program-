## 3) Message Anatomy (Use Every Time)

Every message should follow this sequence:

1. Mirror
   - Name the lived reality as they experience it now.
2. Shift
   - Reframe from confusion/waiting to intentional progress.
3. Mechanism
   - Show the "how" (8 weeks + 4 Habits + guided practice).
4. Proof
   - Show what becomes visible (artifact, habit evidence, confidence signal).
5. Door
   - One low-friction next step: `THINK`.

Template:

```text
[Mirror: "You are here now."]
[Shift: "There is a better way to move."]
[Mechanism: "Here is exactly how K2M works."]
[Proof: "Here is what you will have to show."]
[Door: "Send THINK to [number]."]
```

## 5) Concreteization Engine: Show "Thinking WITH AI" In 60 Seconds

Use this in school visits, videos, WhatsApp voice notes, and live sessions.

### Demo script
1. "Let me show you something in 60 seconds."
2. Prompt A (generic): `Give me a study plan for exams.`
3. Prompt B (thinking):  
   `I have 2 weeks, physics is my weakest subject, I lose focus after 45 minutes, and I can only study after 7pm. Build a realistic plan with 20-minute blocks and one weekly review loop.`
4. Ask: "Which answer sounds like it fits a real person?"
5. Close: "That difference is what we mean by thinking WITH AI."
6. Door: "If you want this as a habit system, send THINK to [number]."

## 9) Student Messaging Playbook (Groups + DMs)

### Do
1. Start from lived feeling ("in-between", "AI feels random").
2. Keep language plain.
3. Make CTA one step only.
4. Use short messages with line breaks.
5. Use social proof carefully ("students like you"), not pressure.

### Do not
1. Open with philosophy.
2. Lead with fear ("you will be left behind").
3. Use technical jargon.
4. Ask for long forms before interest is confirmed.

## 13) Quality Control Checklist (Before Sending Any Message)

1. Is this message written at the correct altitude for this audience?
2. Does it begin with a mirror of real lived experience?
3. Did I explain mechanism (8 weeks + 4 Habits) clearly?
4. Is there a concrete proof outcome?
5. Is there only one CTA (`THINK`)?
6. Is the language plain enough for a first read on mobile?
7. Does tone align with hope (not fear)?
8. Would this message make sense with no prior context?

If any answer is "No", revise before sending.
