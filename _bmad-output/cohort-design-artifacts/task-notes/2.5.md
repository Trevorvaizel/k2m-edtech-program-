**Friday Reflection Flow:**
1. Friday 9:00 AM EAT: Bot posts reflection prompt to current week channel
2. Student uses: /submit-reflection habit-practice:Yes identity_shift:"I learned so much!" proof-of-work:"AI knew I'm interested in game design"
3. Bot saves reflection to database with timestamp
4. Bot confirms submission and informs of Saturday unlock timing

**Saturday Batch Unlock Flow:**
1. Saturday 12:00 PM EAT: batch_unlock_week() runs
2. Database updates: next_week_unlocked=1, unlocked_at=timestamp for all submitted reflections
3. Student.current_week incremented (Week N → Week N+1)
4. Bot posts unlock announcement to current week channel
5. Bot sends Trevor notification with list of incomplete students

**Laggard Handling:**
- Students who didn't submit remain in current week
- They retain POST permissions to current week channel (can catch up)
- They cannot access next week channel until reflection submitted
- Batch unlock skips them (no current_week increment)
- Trevor can manually unlock if needed (illness, emergency)

**Trevor Manual Override:**
- Command: /unlock-week [@student] [week] [reason]
- Use cases: illness, emergency, accelerated pacing, special circumstances
- Manually unlocked flag set to 1 (logged for culture monitoring)
- Trevor's Discord ID and reason tracked in database
