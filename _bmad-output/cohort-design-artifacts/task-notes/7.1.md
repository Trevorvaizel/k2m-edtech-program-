# Task 7.1: Stage 1 Redesign - Reconciled Notes

**Date:** 2026-03-05

**Scope:** `/api/interest`, landing entry flow, refund-policy visibility

## Why This Reconciliation Was Required

A code audit found false-positive completion behavior:

1. `/api/interest` could return `success=true` even when Google Sheets append failed.
2. `/api/interest` could return `success=true` even when email delivery failed.
3. Sheets row append shape was inconsistent with the runtime A-R contract.
4. `#welcome` refund policy pin requirement was documented but not seeded by setup automation.

## Fixes Applied

### Backend (`cis-discord-bot/cis_controller/interest_api_server.py`)

1. Fixed API truthfulness:
   - Return `500` when sheet append fails.
   - Return `502` when email send fails.
   - Do not report success on partial failures.
2. Fixed roster append schema:
   - Canonical A-R row write.
   - Column `L` now carries status (`Lead` or `Waitlisted`).
   - Column `R` stores `invite_code`.
   - `created_at` written at Column `N`.
3. Enrollment behavior retained and hardened:
   - Duplicate email detection.
   - Enrollment-cap waitlist behavior.
4. Invite-link behavior made explicit:
   - Attempts Discord API unique invite creation when configured.
   - Falls back only when `DISCORD_INVITE_FALLBACK_URL` is explicitly set.
   - Fails safely if no invite link can be produced for non-waitlisted users.

### Discord Welcome Seed (`setup_discord_server.py`)

1. Added refund policy section to `WELCOME_CONTENT`.
2. Added `WELCOME_REFUND_PIN` message.
3. Added automation in `post_welcome_content()` to post+pin refund policy if missing.
