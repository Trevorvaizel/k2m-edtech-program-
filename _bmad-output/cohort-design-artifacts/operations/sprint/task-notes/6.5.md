## Implementation Completed

### 1) Tiered model routing in `llm_integration.py`

- Added fast-tier agents: `frame`, `diverge`, `dashboard_summary`, `profession_inference`.
- Added deep-tier agents: `challenge`, `synthesize`, `create-artifact` (`create_artifact` aliases included).
- Extended `get_active_model(provider, agent=None)`:
  - Anthropic: `CLAUDE_HAIKU_MODEL` / `CLAUDE_FAST_MODEL` (fast), `CLAUDE_SONNET_MODEL` / `CLAUDE_MODEL` (deep)
  - OpenAI: `OPENAI_FAST_MODEL` (fast), `OPENAI_REASONING_MODEL` (deep)
  - Zhipu: `GLM_FAST_MODEL` (fast), `GLM_REASONING_MODEL` (deep)
- `call_agent_with_context(...)` now resolves model with `agent=...` for per-command routing.

### 2) Observability model tagging (`model_used`)

- Updated event logging path so `agent_used` writes include:
  - `metadata.provider`
  - `metadata.model`
  - top-level `model_used` column value
- Updated command handlers:
  - `commands/frame.py`
  - `commands/diverge.py`
  - `commands/challenge.py`
  - `commands/synthesize.py`
- Updated store API:
  - `StudentStateStore.log_observability_event(..., model_used=None)`
  - fallback to `metadata["model"]` when explicit arg is absent

### 3) Schema + migration updates

- Added `model_used` column to `observability_events` in:
  - `database/schema.sql`
  - `database/schema_pg.sql`
- Added index `idx_observability_model`.
- Added legacy migration path in `database/store.py` for existing SQLite DBs.

### 4) Dashboard daily model-cost split

- Extended `FacilitatorDashboard._get_api_costs()` to aggregate same-day `agent_used` costs by model.
- `generate_daily_summary(...)` now includes:
  - `Daily by model: modelA=$x.xx, modelB=$y.yy ...`
  - fallback message when no model-tagged usage exists.
