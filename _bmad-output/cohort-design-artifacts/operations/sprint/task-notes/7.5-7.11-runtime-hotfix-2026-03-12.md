# Task 7.5 + 7.11 Runtime Hotfix (2026-03-12)

## Scope
- Production issue triage after repeated dashboard alerts and scheduler failures.
- Focused fixes:
  - Task 7.5 payment-feedback scheduler crash.
  - Task 7.11 401 caller diagnostics.
  - Task 7.6 PG reconnect hardening (documented separately in `7.6-postgresql-connection-recovery-2026-03-12.md`).

## Confirmed Runtime Problems
1. `send_payment_feedback_dms` crash:
   - Error: `ValueError: too many values to unpack (expected 2)`
   - Cause: function iterated `for row_number, row in rows` while `read_roster_rows()` returns raw Sheets matrix (`[header, row1, row2, ...]`), not tuple pairs.

2. Frequent unsigned webhook 401 alerts:
   - Alerts already included source IP, but lacked user-agent context needed for fast source attribution.

3. PostgreSQL stale-connection loop:
   - `SSL connection has been closed unexpectedly` then repeated `connection already closed`.
   - Fixed in Task 7.6 follow-up note.

## Code Changes
- `cis-discord-bot/cis_controller/interest_api_server.py`
  - `send_payment_feedback_dms()` now accepts both supported row iterator shapes:
    - canonical raw Sheets matrix (`enumerate(rows[1:], start=2)`)
    - tuple mocks (`(row_number, row_values)`) used in tests.

- `cis-discord-bot/cis_controller/internal_api_server.py`
  - `_log_401_to_dashboard(...)` now accepts `user_agent` and includes it in dashboard message.
  - All internal webhook handlers now pass request `User-Agent` into 401 dashboard logging.

- Tests:
  - `cis-discord-bot/tests/test_task_75_payment_feedback.py`
    - added test proving raw Sheets matrix shape works without crash.
  - `cis-discord-bot/tests/test_webhook_auth.py`
    - added assertions that 401 dashboard log includes user-agent (default and explicit values).
