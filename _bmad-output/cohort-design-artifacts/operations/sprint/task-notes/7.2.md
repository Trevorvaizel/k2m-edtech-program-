# Task 7.2: Unique Per-Student Discord Invite Links (Reconciled)

**Date:** 2026-03-05  
**Status:** done (reconciled after gap audit)  
**Scope:** invite-code join matching, roster identity bridge, recovery controls

## Gaps Found During Audit

1. `on_member_join` depended on runtime DB rows only; when those rows were missing, Column D in Sheets was not written.
2. `link_student_by_invite()` could not update preloaded `__pending__...` rows.
3. `/recover-member` did not enforce the documented facilitator-only gate.
4. Sprint/docs claims drifted from verifiable behavior.

## Fixes Applied

### Runtime Linkage

**Files:** `cis-discord-bot/main.py`, `cis-discord-bot/cis_controller/interest_api_server.py`

1. Added roster helpers:
   - `find_row_by_invite_code(...)`
   - `link_roster_discord_identity_by_invite_code(...)`
2. `on_member_join` now:
   - Keeps invite-diff matching + `asyncio.Lock`.
   - Falls back to Sheets invite-code linkage and writes Column D (`discord_id|discord_username`) when DB row is not available.
   - Hydrates runtime store from roster data when supported.
3. `/recover-member` now:
   - Requires `@Facilitator`.
   - Syncs Column D through the same invite-code bridge on successful recovery.

### PostgreSQL Linking Robustness

**File:** `cis-discord-bot/database/pg_store.py`

- `link_student_by_invite(...)` now updates rows where `discord_id` is:
  - `NULL`
  - empty string
  - `__pending__...` placeholder
