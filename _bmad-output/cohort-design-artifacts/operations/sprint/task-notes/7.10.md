## Implementation

### 1) Progress-marked API success responses + post-enroll DM

**File:** `cis-discord-bot/cis_controller/interest_api_server.py`

- `/api/interest` success message now returns:
  - `Step 1 of 4 complete - check your email for your Discord invitation.`
- `/api/enroll` success message now returns:
  - `Step 3 of 4 - enrollment complete...`
- Added KIRA post-enroll DM dispatch (fire-and-forget) when linked `discord_id` exists:
  - Includes inline token URL and expiry window
  - Includes Step 3 progress marker
  - Mentions email backup copy
- Updated immediate post-M-Pesa DM text to include Step 4 marker:
  - `Step 4 of 4: Trevor verifies (usually 24 hours).`

### 2) Welcome DM bridge update (N-24 + N-09)

**File:** `cis-discord-bot/main.py`

- Updated linked welcome DM copy in `on_member_join`:
  - Adds `Step 2 complete - here's Step 3`
  - Adds first-timer Discord mobile navigation bridge
  - Keeps value-first onboarding progression intact

### 3) Activation DM webhook path (N-25) + two-message structure

**File:** `cis-discord-bot/cis_controller/internal_api_server.py`

- Added signed endpoint: `POST /api/internal/activation-dm`
- HMAC-authenticated using existing `X-K2M-Signature` flow
- Sends two DMs to activated student:
  1. Celebration-first N-25 style activation DM with one clear action
  2. Stops 1-3 intro moved to next message
- Added route registration and startup log inclusion for activation-dm endpoint
- Added helper builder for activation message formatting with optional session/date context

### 4) Apps Script activation flow now triggers activation DM webhook

**File:** `cis-discord-bot/scripts/apps-script/live/Code.gs`

- Added `callActivationDm_()` signed webhook helper
- Integrated activation-dm call into `activateStudentByRow_()` once prior activation steps pass
- Activation DM webhook failures are appended to row note/errors like other activation failures
