Attempted to trigger Apps Script function execution directly (`clasp run` / `scripts.run`) to emulate a true
Google Sheets `Column L: Pending -> Confirmed` trigger event.

Blocked by:
- `403 PERMISSION_DENIED: The caller does not have permission`
- Script manifest uses `executionApi.access = "MYSELF"` (owner-only execution for API calls).

So, from this account we can deploy/push but cannot invoke owner-only Apps Script execution API functions.
An owner-session manual row edit is still required for the strict single-transaction onEdit proof.
