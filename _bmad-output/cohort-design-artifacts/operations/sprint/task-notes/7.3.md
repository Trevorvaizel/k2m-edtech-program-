# Task 7.3: KIRA DM Architecture Audit (Brevo Pre-Join, KIRA Post-Join)

**Date:** 2026-03-09  
**Status:** done  
**Scope:** Decision B-03 + GAP FIX #2 (Email #2 race condition) + enroll reminder DM hardening

## Gaps Found

1. `/api/enroll` sent Brevo Email #2 immediately, even when Column D had no linked `discord_id`.
2. No retry/backoff window for the `on_member_join` linkage race.
3. No deferred delivery mechanism when linkage was still missing after retries.
4. No scheduler job to drain deferred Email #2 records.
5. No implemented 48h enroll reminder DM path for linked-but-not-enrolled users.
6. DM nudge path lacked explicit guard for invalid/unlinked Discord IDs.

## Fixes Applied

### 1) Email #2 gating + retry/backoff

**File:** `cis-discord-bot/cis_controller/interest_api_server.py`

- Added linked Discord identity extraction for Column D (`discord_id|discord_username`).
- `/api/enroll` now gates Email #2 on linked `discord_id`.
- Added retry polling with exponential backoff (1s, 2s, 4s).
- Added required warning log when linkage is still missing after retries:
  - `discord_id not found after 3 retries for {email}`

### 2) Deferred Email #2 queue

**File:** `cis-discord-bot/cis_controller/interest_api_server.py`

- Added persistent queue storage at:
  - `cis-discord-bot/logs/pending_enroll_email_queue.json`
- Added queue write on `/api/enroll` when linked identity is still unavailable.
- Added queue processing function that sends Email #2 once Column D becomes linked.

### 3) Scheduler integration

**File:** `cis-discord-bot/scheduler/scheduler.py`

- Added hourly queue drain job at `:12` each hour:
  - Processes deferred Email #2 entries and sends when linkage appears.
- Added 18:05 EAT job for 48h enroll nudges:
  - DMs linked Discord members who are still missing enroll profile fields.

### 4) 48h enroll nudge implementation

**File:** `cis-discord-bot/cis_controller/interest_api_server.py`

- Added enroll nudge worker that:
  - Targets only linked Discord IDs.
  - Checks 48h age threshold.
  - Skips already-nudged rows using notes marker.
  - Writes nudge marker to Column M notes.
- Added join timestamp marker write on invite-link identity bridge:
  - `k2m_discord_joined_at=<iso>`

### 5) Explicit DM guard for unlinked identities

**File:** `cis-discord-bot/cis_controller/participation_tracker.py`

- `_send_nudge_dm` now returns early when `discord_id` is non-numeric/unlinked.

## Canonical/Spec Note

- Sprint line item text mentioned “Column E empty” for the 48h enroll nudge condition.
- Canonical enrollment contract stores enroll form completion in fields `F-I` (`zone`, `situation`, `goals`, `emotional_baseline`), while Column E is profession from `/api/interest`.
- Implementation uses missing core enroll fields (`F-I`) as the nudge trigger to avoid false negatives.
