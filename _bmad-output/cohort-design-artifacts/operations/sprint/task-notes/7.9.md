## What Was Implemented

1. **Nightly Stage 1->2 worker**
   - Added `check_stage1_dropoff()` in `cis-discord-bot/cis_controller/interest_api_server.py`.
   - Scans Student Roster and targets only rows where:
     - Column D (Discord identity) is empty
     - Column N (`created_at`) is older than threshold
   - Sends:
     - Email `#1.5` at `>=48h`
     - Email `#1.75` at `>=5 days`
   - Stop condition preserved automatically once Column D is populated.

2. **Idempotency marker (Column V)**
   - Added runtime column constant:
     - `COL_STAGE1_NUDGE_EMAIL_SENT = 21` (Column V)
   - Marker values:
     - `1.5`, `1.75`, or blank
   - Prevents duplicate sends across nightly runs.

3. **Brevo send path for 1.5 / 1.75**
   - Added `send_stage1_dropoff_email()`:
     - Uses templates `BREVO_TEMPLATE_EMAIL_1_5` and `BREVO_TEMPLATE_EMAIL_1_75` when configured.
     - Falls back to inline HTML content if template IDs are absent.

4. **WhatsApp fallback for bounce-like 1.5 failures**
   - Added `send_stage1_whatsapp_nudge()` (Africa's Talking webhook style).
   - Triggered when Email `#1.5` fails with immediate bounce/reject-like response and phone number is present.
   - Uses env-configured endpoint/credentials:
     - `AFRICAS_TALKING_WHATSAPP_URL`
     - `AFRICAS_TALKING_API_KEY`
     - `AFRICAS_TALKING_USERNAME` (optional, defaults to `sandbox`)

5. **Scheduler integration**
   - Added `post_stage1_dropoff_nudges()` in `cis-discord-bot/scheduler/scheduler.py`.
   - Scheduled nightly at **23:20 EAT** with one-run-per-day guard flag.
   - Toggle:
     - `STAGE1_DROPOFF_NUDGE_ENABLED` (default `true`)

6. **Apps Script runtime schema alignment**
   - Updated `cis-discord-bot/scripts/apps-script/live/Code.gs` runtime headers from A:U to A:V.
   - Added `stage1_nudge_email_sent` header and Column V constant in runtime schema tooling.
