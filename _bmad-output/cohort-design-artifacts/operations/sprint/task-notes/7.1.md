## Production Gap Observed

Live submissions showed a schema mismatch in Column E (`profession`):

- Form was sending human labels such as:
  - `Recent Graduate`
  - `Teacher / Educator`
  - `Professional`
- Sheet validation expected canonical enum values:
  - `teacher`
  - `entrepreneur`
  - `university_student`
  - `working_professional`
  - `gap_year_student`
  - `other`

Impact:
- dropdown mismatch noise in sheet ops
- inconsistent data for downstream routing/analytics

---

## Fixes Applied

### 1) Landing form canonical values

**Files:**
- `k2m-landing/src/components/EnrollmentForm/EnrollmentForm.html`
- `k2m-landing/src/components/EnrollmentForm/EnrollmentForm.js`

Changes:
- Replaced option values with canonical enums while preserving user-friendly labels.
- Added frontend normalization guard (`normalizeProfessionInput`) for legacy labels.
- Updated segment preselect mapping (`gap-year`, `teacher`, `professional`) to canonical enum outputs.

### 2) Backend normalization guard

**File:**
- `cis-discord-bot/cis_controller/interest_api_server.py`

Changes:
- Added `_normalize_profession(raw_value)` with:
  - direct canonical pass-through
  - legacy alias mapping (e.g., `Recent Graduate` -> `gap_year_student`)
  - keyword fallback
  - safe default `other`
- `/api/interest` now always writes canonical profession values to Column E.
